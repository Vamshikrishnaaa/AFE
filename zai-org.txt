Zai-org

1. MODEL OVERVIEW
--------------------------------------------------------------------------------
* Model Name: zai-org/GLM-5 (Also updated as GLM-5.1 / GLM-5.2 Series)
* Developer Organization: zai-org (Z.ai)
* Repository URL: https://huggingface.co/zai-org
* Primary Task: Advanced Agentic Engineering, Long-Horizon Task Execution, and Coding
* Architecture: Sparse Attention Transformer utilizing "IndexShare" (reusing the same indexer across every four sparse attention layers to minimize memory/compute costs) along with an enhanced MTP (Multi-Token Prediction) layer for faster speculative decoding.
* Context Window: Up to 1,000,000 tokens (1M Context) stable window.
* License: Apache-2.0 License

2. DATASET DETAILS
--------------------------------------------------------------------------------
* Training Data Mix: 
  - Massive multilingual web-scraped text corpuses.
  - Comprehensive source code repositories across multiple programming languages.
  - Synthetic instruction-following datasets tailored specifically for debugging, terminal workflows, and tool execution.
  - Deep reinforcement learning datasets optimized with Asynchronous Reinforcement Learning for long-session logic alignments.

3. TRAINING PARAMETERS & HYPERPARAMETERS
--------------------------------------------------------------------------------
* Optimization Framework: Asynchronous Reinforcement Learning (RL) combined with Supervised Fine-Tuning (SFT).
* Efficiency Metric: Reduces per-token FLOPs by 2.9× at a 1M context length via IndexShare layer sharing.
* Inference Acceleration: Speculative decoding acceptance length increased by up to 20% via modified MTP configurations.
* Precision Formats: Weights distributed in native BF16, with community quantizations optimized for MXFP4_MOE and Q4_K_M GGUF execution.
* Evaluation Baselines: 
  - Terminal-Bench 2.1: Scores 81.0 (approaching closed frontier architectures).
  - SWE-bench Pro: Scores 62.1 (State-of-the-art open-weights agent performance).

4. INTENDED USE & AUTOMATION
--------------------------------------------------------------------------------
* Primary Applications: 
  - Agentic Engineering: Iterative multi-turn debugging, tool usage, running terminal experiments, and handling ambiguous coding problems.
  - Automated Release Workflows: Used by major open-source ecosystems (like huggingface_hub CI) to aggregate commit logs, generate human-style release notes, and deploy automated PRs.
* Execution Modes:
  - Chat Mode: Instant response, lightweight interactive delivery.
  - Agent Mode: Multi-tool orchestration, continuous execution loops across hundreds of rounds without regression.
